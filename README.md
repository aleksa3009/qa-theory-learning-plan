## üß≠ Learning plan

This repository implements a structured learning plan intended for a Junior QA Engineer. Each topic is designed with clear learning objectives, recommended practical exercises, and examples of interview-style questions for self-assessment. The plan is comprehensive across the core QA domains and framed to support both theoretical understanding and hands-on practice.

Key topical areas:

- **QA fundamentals:** responsibilities of a Junior QA Engineer; differences between SDLC and STLC; test lifecycle.
- **Test artifacts:** test plans, test cases, acceptance criteria, defect reporting and defect lifecycle.
- **Testing types:** functional and non-functional testing; unit, integration, system, UAT, regression strategies.
- **Tools & workflows:** version control with Git; Linux command-line basics relevant to QA; issue tracking and workflow integration (e.g., Jira); logging and debugging with browser DevTools.
- **Databases & APIs:** SQL fundamentals and multi-table queries; API testing concepts and practices; OpenAPI/Swagger usage.
- **Test automation:** JavaScript essentials relevant to automation; Selenium, Cypress, Playwright fundamentals and trade-offs.
- **CI/CD & pipelines:** integrating automated tests into CI systems, pipeline test reporting and artifact management.
- **Performance & security testing:** load/stress testing concepts; OWASP Top 10 awareness; basic security test checks.
- **Quality & standards:** accessibility (WCAG basics), QA metrics, relevant standards and professional guidance (ISO/ISTQB references where applicable).
- **Practical techniques:** mocking and stubbing, flaky-test mitigation, test data management, and using Docker for reproducible environments.

Each topic file will typically include:

- A concise summary and definitions.
- Specific learning objectives.
- Practical exercises or commands to execute.
- Suggested interview questions and quick checks for self-evaluation.
